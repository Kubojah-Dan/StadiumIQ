# StadiumIQ

AI-powered stadium operations assistant for real-time crowd safety and venue decision support.

## Chosen Vertical
**Vertical:** Smart Venue Operations  
**Primary Persona:** Stadium Operations Commander (SOC) managing ingress, queue pressure, and safety response during live events.

## What This Solution Demonstrates
- Smart, dynamic assistant behavior based on live venue context
- Logical decision making from crowd density, alerts, queue load, and connection health
- Practical Google Services integration (Maps + Gemini)
- Real-world architecture for live stadium feed adapters
- Security-first account system with password hardening and optional authenticator 2FA
- Clean, modular code paths for maintainability

## Core Features

### 1) Context-Aware Operations Copilot
- Dashboard calls `/api/assistant/recommendations` with live context.
- If `GOOGLE_GEMINI_API_KEY` is configured, recommendations are generated by **Google Gemini**.
- If not configured, a deterministic rule engine still returns actionable guidance (no broken UX).

### 2) Live Stadium Intelligence Layer
- Dashboard calls `/api/stadium/live?stadiumId=...`.
- Supports:
  - Official feed adapter via env keys like `STADIUMIQ_FEED_IN_NARENDRA_MODI`
  - Fallback live simulator model when official feed is unavailable
  - Stadium image enrichment via Wikipedia summary API
  - Real map integration via Google Maps links/embed

### 3) Real Image + Map on Dashboard
- Stadium card includes:
  - Real venue image when available
  - Embedded map view (Google Maps)
  - Occupancy + gate wait summary

### 4) Security Improvements
- Passwords moved from plaintext storage to **salted PBKDF2-SHA256 hashes**
- Sign-in brute force protection with temporary lockout
- Session payload hardening with expiry validation
- Security headers + CSP in Next.js config
- Optional TOTP 2FA flow with:
  - Setup key copy
  - QR scan option
  - Code verification to enable/disable

## 2FA UX Flow
- Go to `Dashboard -> Settings -> Security & Access`
- Click `Generate 2FA setup key`
- Choose one:
  - Copy manual key into authenticator app
  - Scan QR code
- Enter authenticator code and enable 2FA
- Login now requires password + authenticator code

## Setup

### 1) Install
```bash
npm install
```

### 2) Environment
Copy `.env.example` to `.env` and fill values:
```bash
cp .env.example .env
```

### 3) Start Services
```bash
docker-compose up -d
python -m venv .venv
.venv\Scripts\activate
pip install -r services/ai-engine/requirements.txt
pip install -r services/realtime/requirements.txt
uvicorn services.ai-engine.src.main:app --reload --port 8002
uvicorn services.realtime.app.main:app --reload --port 8001
```

### 4) Start Web App
```bash
npm run dev
```

Open: `http://localhost:3000`

- Password: `Admin@12345`

## Assumptions
- Official stadium telemetry APIs differ by venue; adapter keys are intentionally per-stadium.
- When official feeds are unavailable, fallback telemetry keeps the product demonstrable.
- Gemini is optional; system remains usable with deterministic recommendations.
- QR rendering currently uses Google Chart API in demo mode.
